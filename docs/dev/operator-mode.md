# Operator Mode Guide

This document is for local development and QA operators only. It is not product user documentation and does not grant production operational readiness.

Normal VoidEmpires routes open in product mode. Operator mode is hidden by default and only reveals secondary technical tools for local validation.

## Reveal Mechanisms

Use one of these local, explicit mechanisms:

- One navigation session: add `?operator=1` to the current URL.
- Repeated local work in the same browser profile: run `localStorage.setItem("voidempires.operatorMode", "1")` in the browser console, then reload.
- Return to product mode: run `localStorage.removeItem("voidempires.operatorMode")`, remove any `operator=1` query flag, then reload.

The local storage value must be exactly `"1"` or absent. Do not store credentials, roles, claims, tokens, emails, endpoint URLs, connection strings, raw payloads, or database details in browser storage.

## What Operator Mode May Reveal

Operator mode is a visibility layer. It may reveal technical support surfaces such as:

- `DevelopmentToolsPanel` for manual local materialization or controlled helper actions.
- `DevDiagnosticsPanel` for technical summaries, compact ids, raw payload previews, and backend response context.
- `ActionManifestPanel` for action contract metadata.
- Route-specific technical details such as exact ids, request failures, or backend acceptance details.

Operator mode must not auto-run materialization, queue completion, transfers, resource changes, or other state-changing calls. Any revealed state-changing action still needs the existing explicit review or confirmation flow.

## Route Notes

- `/onboarding`: may reveal returned ids and request failure details.
- `/planet` and `/construction`: may reveal resource/materialization tools and diagnostics.
- `/research` and `/shipyard`: may reveal backend acceptance details, queue/stock payload context, and technical errors.
- `/fleets`: may reveal transfer tooling, action manifests, tactical ids, and raw transfer diagnostics.
- `/galaxy`: may reveal system/planet diagnostics and technical map context.
- `/defenses`, `/ground-army`, `/market`, `/espionage`, `/alliance`, and `/ranking`: may reveal context ids, comparison diagnostics, or raw limitations while keeping the route's read-only/readiness boundary.

## Boundaries

- Operator mode is not authentication, authorization, ownership proof, admin authority, or production login.
- Operator mode must never be inferred from API responses, user names, emails, remote configuration, or backend profile text.
- Product links generated by the app should not preserve `operator=1` during normal navigation.
- Do not use operator mode as evidence of final gameplay, final SQL Server readiness, production auth readiness, final visual acceptance, or final asset readiness.
- Real SQL Server credentials and connection strings remain outside the repository and outside browser storage.
